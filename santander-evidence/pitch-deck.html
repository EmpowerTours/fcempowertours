Proceso de Mint

  • 1. Validate contracts deployed
  • 2. Check WMON balance (150 WMON)
  • 3. Auto-wrap MON WMON si necesario
  • 4. Approve WMON spending
  • 5. Call mintFor(beneficiary, fid, countryCode)
  • 6. NFT llega a wallet del usuario

Post-Mint Pipeline

  • Envio indexa PassportMinted event
  • Dynamic SVG generado con stamps
  • AI genera sellos circulares en check-in
  • Passport renderizado en /api/passport/image/[id]
  • Stamps almacenados en IPFS (Pinata)
  • Metadata on-chain: country, fid, verified

Seguridad & Infraestructura

Smart Contract Security

  • OpenZeppelin: ReentrancyGuard, Ownable, Pausable
  • SafeERC20 para transferencias de tokens
  • onlyAuthorizedMinter modifier
  • Rate limiting: 24h cooldown entre mints
  • Input validation: country codes, amounts
  • Timelock en operaciones criticas

Backend Security

  • Rate limiting por IP y wallet address
  • Delegaciones: permisos granulares, 24h expiry
  • WMON approval limitada (no unlimited)
  • Sanitizacion de inputs (XSS, injection)
  • Admin actions requieren signature
  • Anti-bot: flagging comunitario
